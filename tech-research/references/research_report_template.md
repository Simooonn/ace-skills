# Research Report Templates

## Template A: Overview Report (概览报告)

```markdown
# {Topic} 深度调研报告
> 调研日期: {date}
> 调研模式: {Quick/Standard/Deep/UltraDeep}

## TL;DR
One paragraph summary of key findings.

## 核心问题解答
(If user asked specific questions, answer them here first)

## 核心概念
What is it? What problem does it solve?

## 工作原理
How it works, with diagrams if helpful.

## 框架/工具对比

| 项目 | 定位 | 优势 | 劣势 | 适用场景 | GitHub Stars | 最近更新 |
|---|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... | ... |

## 最新动态 (2025-2026)
Latest developments from real-time search results.

## 适用场景 / 不适用场景
When to use and when NOT to use.

## 快速上手路径
Beginner-friendly getting started path.

## 排除项目
| 项目 | 排除原因 |
|---|---|
| ... | ... |

## 参考来源
Full citations with URLs.

## 附录 A: 证据表
(Standard/Deep/UltraDeep only)

---
Research Metadata:
- Mode: {mode}
- Sources consulted: {count}
- Search passes: {count}
- Tools used: {list}
- Evidence confidence: {summary}
- Date: {date}
---
```

## Template B: Hands-on Guide (上手指南)

```markdown
# {Topic} 上手指南
> 日期: {date}
> 前置条件: {prerequisites}

## 概述
What this guide covers and expected outcome.

## 环境要求
- OS, runtime, dependencies

## 方案 A: {Approach Name}

### A-1: 环境准备
Exact commands and expected output.

### A-2: 安装配置
Step-by-step with code blocks.

### A-3: 核心使用
Main workflow with examples.

### A-4: 验证测试
How to verify it works.

## 方案 B: {Approach Name}

### B-1: 环境准备
(Self-contained — repeat shared config here)

### B-2: 安装配置
...

### B-3: 核心使用
...

### B-4: 验证测试
...

## 常见问题排查
FAQ and troubleshooting.

## 进阶配置
Advanced options for power users.
```

## Structure Rules

- Overview report: use Template A
- Hands-on guide: use Template B
- Both templates output in Chinese Simplified by default
- Code snippets, CLI commands, library names: keep English
- Each approach in Template B must be fully self-contained
- Shared configuration must be repeated in each approach section
