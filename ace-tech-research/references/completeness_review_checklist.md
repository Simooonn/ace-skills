# Completeness Review Checklist

Verify the draft meets all requirements before delivery.

## Structure and Format

- [ ] All required sections present and ordered per report spec
- [ ] Headings match the format contract
- [ ] Length targets met per section
- [ ] Language is Chinese Simplified with English technical terms
- [ ] Language is consistent throughout

## Evidence and Citations

- [ ] Every claim has a citation with URL
- [ ] Every numeric claim has at least one Tier A or B source
- [ ] Conflicting sources are explicitly noted
- [ ] Evidence table maps claims to sources
- [ ] No orphan claims without citations

## Content Quality

- [ ] Findings answer the research questions
- [ ] Must-Answer Questions addressed at top of report (if any)
- [ ] Recommendations are tied to evidence
- [ ] Limitations and uncertainty are documented
- [ ] Terminology is consistent

## Activity Screening

- [ ] All projects in core comparison pass activity screening criteria
- [ ] Excluded projects table is present with reasons
- [ ] No unmaintained projects in core comparison

## Deliverable Completeness

- [ ] Deployable tech → both overview report AND hands-on guide produced
- [ ] Pure concepts → single overview report is sufficient
- [ ] Hands-on guide approaches are self-contained pipelines
- [ ] Each approach includes exact commands and expected output

## Final Checks

- [ ] Dates and time ranges are consistent
- [ ] No unsupported claims
- [ ] No duplicate or conflicting statements
- [ ] Files placed in correct `research/{topic}/` directory
